# Сортировка слиянием для односвязного списка

## Суть задачи

Нужно реализовать merge sort не для массива, а для собственного односвязного списка.

В проекте реализуются:
- конструктор списка из `initializer_list`
- функция слияния двух уже отсортированных списков
- рекурсивная сортировка слиянием

## Структура

Основные файлы:
- `merge-sort/merge-sort.hpp` — описание структур `ListElement` и `List`
- `merge-sort/merge-sort.cpp` — реализация конструктора, `merge` и `mergesort`
- `merge-sort/test.cpp` — тесты

## Как работает решение

### Конструктор `List::List`

Конструктор проходит по `initializer_list` и дописывает элементы в конец списка.

Для этого:
- первый элемент становится `head`
- указатель `tail` хранит конец текущего списка
- каждый новый узел добавляется в `tail->next`

### Функция `merge`

Слияние двух отсортированных списков происходит без копирования значений:

1. Сравниваются головы списков `a` и `b`.
2. Меньший узел переносится в результирующий список.
3. Перенос реализован через `std::move`, то есть перекидывается владение `unique_ptr`.
4. Когда один из списков заканчивается, остаток второго просто присоединяется к результату.

### Функция `mergesort`

Сортировка работает по классической схеме merge sort:

1. Если список пустой или из одного элемента, сортировка не нужна.
2. Середина находится техникой slow/fast pointers.
3. Список разбивается на две половины.
4. Обе половины сортируются рекурсивно.
5. Результат собирается через `merge`.

## Какие алгоритмы и подходы применены

- merge sort
- поиск середины списка двумя указателями
- перенос узлов через `std::move`
- управление памятью через `std::unique_ptr`

## Сложность

- по времени: `O(N log N)`
- по памяти: `O(log N)` на глубину рекурсии

## Как собрать и запустить

```bash
cmake -S . -B build
cmake --build build
ctest --test-dir build
```

## Что важно в этой реализации

- не используются явные `new` и `delete`
- элементы не копируются лишний раз
- логика переноса головы вынесена в отдельную функцию `head_moving`
