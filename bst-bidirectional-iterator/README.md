# Двунаправленный итератор для BST

## Суть задачи

Нужно реализовать итератор двоичного дерева поиска, который поддерживает:

- переход к следующему элементу `++`
- переход к предыдущему элементу `--`
- начало обхода `begin()`
- конец обхода `end()`

Порядок обхода — инфиксный, то есть элементы дерева возвращаются по возрастанию.

## Реализация

Основные файлы:
- `tree-iterator/tree.hpp` — структура дерева и итератора
- `tree-iterator/tree.cpp` — реализация `operator++`, `operator--`, `begin`, `end`
- `tree-iterator/test.cpp` — тесты

## Как работает итератор

### `begin()`

Возвращает самый левый узел дерева, то есть минимальный элемент.

### `end()`

Возвращает специальный итератор с `nullptr`, обозначающий позицию "после последнего".

### `operator++`

Алгоритм поиска следующего элемента:

1. Если у текущего узла есть правое поддерево, берётся самый левый элемент в этом поддереве.
2. Иначе поднимаемся вверх по указателям `up`, пока текущий узел остаётся правым ребёнком.
3. Первый предок, для которого мы пришли из левого поддерева, и есть следующий элемент.

### `operator--`

Алгоритм поиска предыдущего элемента:

1. Если итератор стоит на `end()`, нужно перейти к максимальному элементу дерева.
2. Если у узла есть левое поддерево, берётся самый правый элемент в нём.
3. Иначе поднимаемся вверх, пока текущий узел остаётся левым ребёнком.
4. Первый подходящий предок является предыдущим элементом.

## Какие алгоритмы и подходы применены

- inorder successor / predecessor
- подъём по родительским указателям
- поиск левого и правого крайних узлов

## Сложность

- переход `++` или `--`: `O(h)` в худшем случае
- дополнительная память: `O(1)`

где `h` — высота дерева.

## Особенность данной реализации

Для корректной работы `--end()` используется вспомогательный статический указатель `current_root`, который запоминает корень дерева при вызове `begin()` и `end()`.
